How To Read It

1. What developers should install

@dexterai/x402-discovery is the clearest install path for developers who want an agent to find, inspect, and pay for x402 APIs.

2. What the brand is called

OpenDexter is the product brand. @dexterai/opendexter is the brand-facing package identity for the same local MCP surface.

3. What builders use instead

@dexterai/x402 is the SDK. It is for teams building x402 into their own apps, clients, servers, and products, rather than installing a ready-made marketplace client.

4. Why discovery still has a wallet tool

The wallet is supporting infrastructure for the local flow. Discovery gets you to the API; the wallet lets the local MCP actually pay and execute.

5. The shortest positioning

x402-discovery = use x402 APIs
x402 = build x402 into your own product

Recommended public phrasing: “Install @dexterai/x402-discovery to search, check, and pay for x402 APIs from your MCP client. Use @dexterai/x402 if you’re building x402 into your own software.”
